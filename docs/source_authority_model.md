# Source Authority Model

This file defines how source materials should be classified and used.

## Authority Levels

| Level | Definition | Use |
|---|---|---|
| Authoritative | Controlling input or approved project position | Can guide decisions unless contradicted by later human decision |
| Supporting | Useful and relevant but not controlling | Can support recommendations |
| Reference-only | Background context | Can inform thinking but should not drive decisions |
| Historical | Prior thinking that may be stale | Use cautiously and flag conflicts |
| Speculative | Brainstorming, exploratory, or unapproved concepts | Do not treat as fact or decision |
| Unknown | Insufficient context to classify | Ask for human review |

## Source Use Rules

- Do not treat all sources as equally authoritative.
- Preserve traceability to source files.
- If sources conflict, flag the conflict.
- If a source is stale, classify it as historical unless the user says otherwise.
- If a source contains speculation, classify it as speculative.
- If source authority is unclear, ask for human review.
